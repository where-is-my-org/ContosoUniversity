# 場景一：舊有程式碼轉規格文件

## 目標
從現有的遺留程式碼分析需求，並將其轉換為結構化的規格文件，對於維護遺留系統、重構專案或理解未有文件記錄的功能至關重要。

## 需求
- 閱讀遺留程式碼
- 建立規格文件（包括功能識別與需求抽取）
- 所有規格文件將被放入 Specs/ 資料夾

## 資料夾結構
- 確保專案結構清晰，便於維護與擴展
- 建議資料夾結構如下：
```
/ContosoUniversity
│── /ContosoUniversity              # 遺留之程式碼
│── /Specs                          # 規格文件
│── /Lab                            # 操作練習文件
│── README.md                       # 專案說明文件
|── .gitignore
```
